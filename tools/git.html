# Git笔记

# 获取和修改git提交者信息

电脑上安装好git后首先修改git提交者信息，上传到本地库后可查询到提交者的信息，在进行远程库操作时也可以查询到其他提交者信息

修改信息

git config --global user.name xxx
git config --global user.email xxx

查询信息

git config user.name
git config user.email

# git当中的角色

  1. 工作树（代码区）

  2. 索引（暂存区）

  3. 本地数据库

# add命令操作

将新增文件或修改文件添加到索引

将当前工作树目录中所有新增或修改文件添加到索引：

git add .

将指定新增或修改文件添加到索引：

git add ./text.txt

多个文件（以空格符将文件相对地址分开）：

git add text.txt ./test2.txt

删除索引相关文件：

git rm --cache 文件名1 文件名2 .

# pull命令操作

进行拉取(Pull) 操作，就是从远程数据库下载最近的变更日志，并覆盖自己本地数据库的相关内容。所以在执行pull命令前请确保未执行commit，确保本地变更日志在未pull前不存在远程库不存在的变更日志记录

基本操作：

git pull 远程库链接/别名 远程库分支名

# commit命令操作

将索引中的资源提交到本地数据库

git commit -m 'xxxx'

# remote命令操作

主要用于为远程库链接取别名和查看当前本地库中远程库别名列表

取别名

git remote add 别名 远程库链接

当不写别名时，别名默认为origin

git remote add 远程库链接

当git clone克隆远程库代码到本地库时，克隆远程库链接默认取别名为origin

# push命令操作

将提交到本地库的变更日志记录新增到远程库变更日志信息及修改或新增的文件与目录覆盖远程库相应的文件与目录

注意点：确保push时本地库比远程库版本新！

push基本推送操作：

当本地库与要提交到的远程库分支名相同时

git push 远程库链接/别名 本地库分支名

举例：
git push origin master
相当于(':'前为本地库分支名，后为远程库分支名)：
git push origin master:master

当本地库与要提交到的远程库分支名不同时

git push origin develop:master

指定-u选项，下一次推送可省略远程库分支名（用于本地库分支名与远程库分支名一致时）

git push -u 远程库链接/别名 本地库分支名

当在clone（克隆）的数据库目录执行推送时，可以省略数据库和分支名称。

git push

当前git绑定github账户或者gitee账户无远程库访问权限，需重新绑定时，可打开windows凭据管理器删除指定账户凭据

# clone命令操作

使用clone指令可以复制远程库文件与变更日志到本地，并创建本地库，在<repository>指定远程数据库的URL， 在<directory>指定新目录的名称。

git clone 远程库链接 指定存放的新目录名称（可选，默认为远程库项目名）

当clone远程库指定分支资源到本地库

git clone --branch 分支名

# reset命令操作

在reset可以遗弃不再使用的提交。执行遗弃时，需要根据影响的范围而指定不同的模式，可以指定是否复原索引或工作树的内容。

模式名称HEAD的位置索引工作树
soft修改不修改不修改
mixed修改修改不修改
hard修改修改修改

基本操作：

不保留工作树代码

#回退上一个版本
git reset --hard HEAD~

#回退上上一个版本
git reset --hard HEAD~~

保留工作树代码

#回退上一个版本
git reset --mixed HEAD~

#回退上上一个版本
git reset --mixed HEAD~~

# git 解决冲突

冲突原因：

  1. 本地库已提交一条变更记录，但未与远程库同步更新，导致拉取时远程库更新的代码与提交变更记录中的代码发生冲突

  2. 本地库代码在工作树，但未与远程库同步更新，想要拉取远程库更新的代码但与工作树代码起冲突，可能会发生覆盖。

解决冲突：

  1. 先pull,提示需要merge,然后在vscode或者任一编辑器进行代码合并。合并完根据git status提示的内容，重新 git add ，git commit -m 'xxx' ，git push

  2. 方式一：先将工作树代码提交到本地库，然后操作同上；方法二：git stash将本地最新变更日志后编辑的代码或者索引的代码进行保存，git pull拉取至远程库最新变更日志，再git stash pop 将代码拿出来并进行merge

# 分支

在版本控制过程中，同时推进多个任务，为每个任务，我们就可以创建每个任务的单独分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线分支的运行。对于初学者而言，分支可以简单理解为副本，一个分支就是一个单独的副本。（分支底层其实也是指针的引用）

简单理解：分支就是将主线分支的文件拷贝一个副本，程序员在分支中对副本进行操作，不会影响到主线分支的文件。等到程序员对分支副本的操作完成后，将副本文件的内容与主分支文件的内容合并

# 分支操作

查看分支：

#指令：
git branch -v

绿色分支名代表当前所在分支，当前分支指向的版本是25bc91a

分支创建：

#指令：
git branch 分支名

切换分支：

#指令：
git checkout 分支名

切换分支后，我们可以重复修改文件步骤：修改文件副本→添加到暂存区→提交到本地库

合并分支：

切换到主线分支，将支线分支内容进行合并

#指令：
git merge 支线分支名

解决合并冲突：

冲突产生的原因：

合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。Git 无法替我们决定使用哪一个。必须人为决定新代码内容。

解决：

编辑有冲突的文件，删除特殊符号，决定要使用的内容

提交本地库时不能带文件名
